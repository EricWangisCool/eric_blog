高效開發必備：Git 基礎使用流程指南

發佈日期：2026-05-27 | 分類：版本控制

Git 是現代軟體開發中不可或缺的版本控制系統。無論是個人開發還是團隊協作，掌握 Git 的核心工作流（Workflow）都能讓你的程式碼管理變得井然有序。本文將用最直覺的步驟，帶你走過一遍標準的 Git 使用流程。

在進入步驟之前，我們需要先理解 Git 的「三個在地端區域」：工作區 (Working Directory)、暫存區 (Staging Area) 以及 儲存庫 (Repository)。接下來的流程就是圍繞著這三個區域進行。

  • 步驟一：初始化或複製專案

    如果你要從零開始一個新專案，可以在專案目錄下初始化 Git：

    git init

    若是加入現有的團隊專案，則是從遠端儲存庫（如 GitHub / GitLab）複製下來：

    git clone <遠端儲存庫 URL>
  • 步驟二：建立分支 (Branching)

    為了保持主線（通常是 main 或 master）的穩定，開發新功能或修復 Bug 時，強烈建議建立獨立的分支：

    # 建立並切換到新分支
git checkout -b feature/login-page
  • 步驟三：修改程式碼並加入暫存區 (Add)

    當你在工作區完成程式碼的撰寫或修改後，可以使用以下指令將檔案加入暫存區，準備進行版本標記：

    # 加入特定檔案
git add src/main/java/Main.java

# 或加入所有變更檔案
git add .
  • 步驟四：提交變更 (Commit)

    將暫存區的檔案正式存入地端儲存庫，並撰寫有意義的 Commit Message，記錄這次修改的目的：

    git commit -m "feat: implement user login authentication"
  • 步驟五：推送至遠端儲存庫 (Push)

    地端紀錄完成後，將你的本地分支推送到遠端伺服器，讓團隊成員看見你的變更：

    git push origin feature/login-page
  • 步驟六：建立 Pull Request (PR) 與合併

    最後，在 GitHub 等平台上發起 Pull Request。經過團隊 Code Review 並且跑完 CI/CD 自動化測試（例如安全掃描、單元測試）確認無誤後，即可將程式碼安全地合併（Merge）回主分支！

小結

簡單來說，Git 的日常循環就是：修改程式碼 ➔ git add ➔ git commit ➔ git push。養成良好的 Git 使用習慣，不僅能保護自己的程式碼資產，更能讓團隊協作更加順暢。
